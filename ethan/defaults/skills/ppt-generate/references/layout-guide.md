# 版式布局指南（画布 1000 × 562.5）

> 给 LLM 的排版坐标参考。所有数值单位 px，原点左上。
> 目标品质：**华为/咨询级汇报 PPT**——观点先行、信息密集、结构严谨、总览与细节互相锚定。

## 全局网格

- 页边距：左右 60，上 36，下 30。
- 安全红线：任何元素 `left+width ≤ 940`、`top+height ≤ 540`（页脚除外）。
- 页脚：`left=60, top=530, width=880, height=20`（textType=footer）：左侧资料来源，右侧页码。

## 咨询式页面骨架（每个 content 页都按此搭）

自上而下五层，缺一不可（②③④是咨询品质的核心）：

| 层 | 位置 | 内容 |
|---|---|---|
| ① kicker 章节标签 | left=60, top=36, height=18 | `02 · 核心机制`，fontSize 11-12，主题色，字间距略宽 |
| ② action title 观点标题 | left=60, top=58, width=760, height=48 | **一句完整结论**，fontSize 24-26 bold，最多 2 行 |
| ③ 章节导航 | right 侧，top=40 | 4 个小圆点（ellipse 14×14，x 从 846 起步进 24）：当前章填主题色，其余填边框色 |
| 分隔线 | line [60,116]→[940,116] | 宽 1.5，边框色 |
| ④ 内容区 | top=132 至 500 | **2-4 个信息区块**，见下方版式库 |
| ⑤ takeaway 栏 | left=60, top=505, width=880, height=24 | 一行加粗结论（主题色），或资料来源 textType=notes |

### action title 写作规则（最重要的一条）

标题必须是**带主语、动词、结论的完整句**，读者只看标题就能复述全 deck 的主线。

| ❌ 名词短语（禁止） | ✅ 观点句 |
|---|---|
| Self-Attention 机制 | 自注意力用一次矩阵运算替代 n 步循环，任意位置间路径长度降为 O(1) |
| 位置编码 | 注意力本身不知顺序——正弦编码注入多频位置信号，RoPE 让相对位置可外推 |
| 代表模型 | Decoder-only 凭借生成范式统一生态，LLaMA-3 已 scale 到 405B |

- 长度：≤ 30 字/行，允许 2 行；**尽量带一个数字或机制名**。
- cover/end 页除外（可用短标题）。

## 信息密度标准（丰富 ≠ 字多）

- **一页一个重点**：这页要让读者记住的那句话 = action title；所有区块都为它服务，无关内容一律不放。
- **图为主、字为注**：优先用架构图/示意图/图表/图标卡片承载信息，文字只做注解和结论。纯文字 bullet 页是最次选择，能用图说清楚就不写字。
- 每页内容区 **2-4 个区块**，形态错开（图+卡片、公式+注释、表格+结论栏），不要 4 个纯文字块。
- 每条 bullet 必须含至少一项：**具体数字 / 机制解释（因为…所以…）/ 对比（A vs B）**。
  - ❌「多头注意力效果好」✅「8 个头各管 64 维，分别捕获语法与指代」
- 整页文字量 **80-150 字**（不含表格内文字）；低于 60 字说明没讲透，超过 180 字先拆成图再考虑分页。
- 禁词清单：「非常重要」「效果显著」「广泛应用」「具有重要意义」——没有数字支撑的形容词一律删掉。
- 表格/图表页必须给**结论栏**（图表讲什么）和**来源行**（数据出处）。

## 整体 ↔ 细节锚定（总-分-总结构）

咨询级 deck 的读者永远不迷路，靠四个机制：

1. **总览页先行**：机制/架构类主题，在细节页之前放一页全景图（架构图/框架图），图上用编号圆圈（ellipse 26×26 主题色底白字）给每个模块标 ①②③④。
2. **细节页带锚点**：每个细节页在标题右侧或 kicker 处放**同编号圆圈**，读者知道在讲总览的哪一块。编号必须全局一致。
3. **章节导航点**：每页右上角的圆点标记当前章节（见骨架③）。
4. **回扣页收尾**：细节讲完后，再放一次同一幅总览图，每个模块旁边配一句「现在能讲透的」总结——总分总闭环。

## 页面衔接（让 deck 读起来是一个整体）

逐页生成最大的风险是页与页各说各话。三个衔接手段，在大纲阶段定好、逐页生成时兑现：

1. **承上钩子**：内容页开头用一句话接住上页——「上一页的公式里，Q/K/V 三个矩阵从哪来？这页拆开看」。适合写在副标题或第一个内容区块。
2. **启下钩子**：页尾 takeaway 栏或最后一个区块给下页埋点——「先记住 softmax 这一步，第 9 页我们亲手算一遍」。**页码必须写准**，生成目标页时回头核对。
3. **章节导航一致性**：kicker 格式全 deck 统一（如 `第 01 章 · 核心直觉`），导航点数量=章节数，当前章填主题色——逐页生成时照抄上一页的格式串，不要每页重新发明。

生成某页时用到大纲里别人的钩子，就在该页文案里显式引用（页码/章节名）；复审时检查每个钩子都兑现了，没有空挂的「详见后文」。

## Design system 定制（一个 deck 一套视觉语言）

预设主题是兜底，正式 deck 在 Step 2 定制专属设计语言，规则在生成过程中必须自洽：

- **命名与语义**：给设计语言起个能指导决策的名字（「地图策略」→ 路线、坐标、里程碑装饰；「蓝图」→ 描边、网格、标注线）。名字不是装饰，后面每个装饰元素都问「符合这个语言吗」。
- **色板纪律**：主色 1 个 + 强调色 ≤1 个 + 语义色若干（同一语义全 deck 同色，如「输入蓝、权重橙、输出绿」）。浅色背景卡片用主色 8-12% 透明度色。
- **装饰语法统一**：编号圆圈、章节导航点、分隔线、卡片形态（圆角大小、描边还是投影）——全 deck 只用一套，逐页照抄坐标与样式。
- **字体配对**：按 references/fonts.md 定中文 + 西文，写进 theme 的 fontName / latinFontName。
- **落成内联 theme**：把上面全部决定写进 deck.json 的内联 theme 对象（含 typography 字号阶梯），页面元素尽量用 textType 吃主题默认值，少在 run 级逐个覆盖。

## 页内构件清单（内容翔实的落地方式）

「翔实」不是字多，是**一页里多个形态各异的构件各司其职**。搭内容页时从下表选 2-4 个构件组合，形态必须错开：

| 构件 | 形态 | 职责 |
|---|---|---|
| 观点定义块 | 左侧竖色条 + 定义句（shape rect 2-3px 宽 + text） | 给可复述的定义/结论 |
| 公式 + 出处 | latex 元素 + notes 小字（作者、年份、式号） | 硬核内容锚点 |
| 类比/对照表 | table 元素（角色 × 含义 × 在系统里做什么） | 降理解门槛 |
| 流程/机制图 | roundRect 模块 + line 箭头，箭头旁标注传递内容 | 把文字过程变成图 |
| 数据图表 + 结论栏 | chart + 右侧「关键结论」栏 | 用数据说话 |
| 互动提问块 | 底部通栏（主题色 8% 底）+「课堂快问/思考题」 | 课件必备，让读者停下来 |
| takeaway 栏 | 骨架⑤，一行加粗结论 | 本页唯一要被记住的话 |

- 互动提问块适合「先提问、后文某页揭晓答案」的跨页钩子——在提问里写「第 N 页我们亲手算一遍」，第 N 页开头必须兑现。
- 同一页不要出现两个同类构件（两个表格、两张饼图）。

## 叙事结构（Step 1 大纲阶段就要定）

- **金字塔原理**：结论先行。整 deck 的核心结论 = 封面副标题/第一章；每章结论 = action title；每页论据 MECE（不重不漏）。
- **SCQA 开场**：动机页按 情境(Situation)→冲突(Complication)→问题(Question)→答案(Answer) 组织，答案就是本 deck 主题。
- 每页在大纲里就要写明：action title、3-4 个支撑要点（含数字）、页面角色（总览/细节①/对比/结论）、与总览图的锚点编号。

## 各 slideType 版式

### cover 封面

| 元素 | 位置 | 说明 |
|---|---|---|
| 主标题 | left=100, top=190, width=800, height=80 | 居中，fontSize 40-48，bold |
| 副标题 | left=100, top=285, width=800, height=36 | 居中，textType=subtitle，**写全 deck 核心结论句** |
| 装饰色条 | left=460, top=160, width=80, height=6 | 主题色 roundRect |
| 汇报人/日期 | left=100, top=340, width=800, height=28 | 居中，textType=notes |

变体：左文右图封面 —— 文字区 left=80,width=440；主图 left=560,top=0,width=440,height=562.5（全高出血，imageType=pageFigure）。

### contents 目录

- 标题同正文页标题条。
- 2×2 卡片网格：卡片 width=420, height=140；x = 60 / 520；y = 140 / 300。
- 每条 = itemNumber + itemTitle + **一行本章核心结论**（不是内容罗列，是这章要让读者记住的话）。
- 条目 ≤6 个。

### transition 过渡页

- 咨询式 deck 用 kicker + 导航点替代独立过渡页；页数充裕（>12 页）或用户明确要求时才用。
- partNumber：left=80, top=180, width=200, height=120，fontSize 60+，主题色。
- 章节标题：left=80, top=310, width=700, height=60，fontSize 32-36 bold，**同样是观点句**。

### content 正文页版式库（内容区 top=132 起）

**三/四卡片**：卡片 width=270（三栏 x=60/365/670）或 width=205（四栏 x=60/285/510/735），
top=140, height=280；卡片 = roundRect + 图标(48×48) + itemTitle + 2-3 行机制说明。
卡片正文必须落到机制/数字，不写口号。

**左右论点-证据**：左侧论点/公式/文字 left=60, width=430；右侧证据（图表/表格/示意图）left=530, width=410。
适合「公式 + 直觉解释」「观点 + 数据图」。

**图表页**：图/表占 left=60, top=140, width=560, height=340；右侧结论栏 left=660, top=140, width=280
（itemTitle「关键结论」+ 2-3 条含数字的 item）；来源行 left=60, top=505。

**对比表页**：表格 left=60, top=140, width=880；下方 2-3 条 bullet 讲表格读出的趋势（不是复述表格）。
表体行高 ≥40，列数 ≤5，单元格文字 ≤16 字。

**大数字页**：2-4 个大数字卡横排，数字 fontSize 44-56 bold 主题色 + 单位 + 下方 2 行说明。
数字必须有出处或推导逻辑。

**流程/架构图页**：用 roundRect 模块 + line 箭头搭原生矢量图；模块按数据流向自下而上或自左而右；
关键模块挂编号圆圈（①②③④）供细节页锚定；箭头旁标注传递的内容（如 "K, V"）。

**公式页**：公式元素居中 width=700-880，fontSize 24-32；
上方一段「为什么需要它」，下方分块解释每个符号的含义 + 一条工程 implications。
公式不是装饰，符号必须逐一解释。

**练习/反馈页（课件场景）**：题干块（通栏卡片，left=60, top=140, width=880, height=120，主题色 8% 底）
写题目与要求；下方 2-3 个选项/提示卡片（三卡片坐标，height=180）；
底部 takeaway 栏写「先自己作答，第 N 页揭晓」式的钩子。
反馈/答案页与练习页**同版式同坐标**复现题目，答案区块用强调色描边，
并回扣提问页页码（「回到第 N 页的问题」）。

### end 结尾页

- 「谢谢观看 / Q&A」居中 fontSize 40 bold，top=230。
- 下方一行参考文献/延伸阅读 textType=notes（咨询级 deck 给出处）。
- 与封面同色系渐变背景收尾。

## 配色纪律

- 一页内除主题色外最多 1 个强调色；themeColors[0] 用于标题强调/图表主系列/编号圆圈。
- **主色加灰调更雅致**：高饱和纯色（纯钴蓝 #2B5CE6 这类）大面积使用显「硬」。正式 deck 的主色往灰里调一档（莫兰迪式），如皇家蓝加灰 `#4A5FA8`、墨绿加灰 `#4A7A6A`——低饱和主色 + 纸感浅底是咨询级的安全牌。
- **大面积填充优先「深色字 + 浅色底」**：流程步骤条、结论栏、表头这类大块面，用主色 8-12% 的浅底 + 深蓝/深灰字，比「白字 + 深底大色块」雅致且易读。「白字 + 深底」只留给小面积徽章（编号圆圈、小标签）和封面/结尾的整页背景。
- 浅色背景上的卡片 fill 用主题色的 8-12% 透明度色（如 #1E40AF → #EFF6FF）。
- 正文颜色直接用主题 fontColor，不要随手发明灰色。
- 深色主题（dark-tech）：卡片 fill 用 #1E293B，文字 #E2E8F0，强调 #38BDF8。
- 同一语义同一颜色：全 deck 中「Encoder 蓝、Decoder 粉、警示黄」这类语义色要前后一致。

## 公式与符号纪律（伪公式零容忍）

- **展示型公式一律 `latex` 元素**（默认 OMML：真根号/真分式，PowerPoint/WPS 里可再编辑；Keynote 交付加 `"engine": "runs"`）：定义式、机制图流程框里的表达式、任何含分式/根号/上下标的展示内容。流程框里需要公式时拆成「形状（无底纹文字）+ label 文本 + latex 元素」三层，不要把表达式写进形状文本。
- **行内变量用 `sub`/`sup` run**：正文句子里提到 d_k、x_i 等变量时拆 run 做真下标（`{"text":"d"}` + `{"text":"k","sub":true}`），禁止裸写 `d_k`、`x_i`。
- **行内根号加组合上划线**：句子/形状标签里绕不开的 `√d`、`√512`，给被开方字符加 U+0305 组合上划线（`√d̅`、`√5̅1̅2̅`），裸 √ 没有横线是廉价感来源。
- **禁止裸文本伪公式**：`Q·Kᵀ ÷ √d_k` 这种写法里下划线不会变下标、√ 没有横线，是最显眼的廉价感来源——识别到 `_k`、`√d_k`、`\sqrt` 字面量一律按上面三条返修。
- 形状内嵌文本（编号圆圈、步骤条、按钮）必须 `text.align: "middle"`，否则文字顶锚定，小徽章里的字符会视觉偏下。

## 字数纪律（下限防空洞，上限防溢出）

- action title ≤ 30 字/行 × 2 行；itemTitle ≤ 16 字；bullet 单行 ≤ 42 字，允许 2 行。
- 卡片正文 ≤ 3 行 × 22 字；整页 **80-150 字**（不含表格/公式内文字）。
- 多构件页（含表格 + 流程图 + 提问块的组合页）可到 200 字，但前提是字都分布在各自构件的独立区域内，不是连续段落。
- 超过上限先拆成图示、再考虑分页；低于下限必须补机制/数字，**不允许靠放大字号撑版面**。
