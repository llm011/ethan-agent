# Team Manager

团队人员管理技能，覆盖两大核心场景：**绩效管理**、**任务委派**。

> 时间线和定时任务管理已拆分到 `schedule-manager` 技能。本技能专注于**人**的管理。

## 适用人群

管理 5-20 人技术团队的 leader / 管理者，希望 Agent 辅助日常管理、减少重复跟进工作。

## 两大场景

### 1. 绩效管理

平时持续收集成员工作事件，绩效季自动汇总并生成评估建议。

**信息来源**：
- 手动记录 — 随时跟 Agent 说，自动归类存储
- 文档分析 — 转发下属文档时，提取亮点数据、架构思路、业务价值
- CR 统计 — 按人汇总代码产出、质量问题、需求归属
- 群消息提取 — 通过定时任务（`schedule-manager` 创建）扫描指定群，识别候选事件

### 2. 任务委派

将任务分配给成员，自动拆解阶段、创建飞书任务、设置 checkpoint 提醒。

**流程**：用户描述任务 → Agent 拆分阶段 → 确认 DDL → 创建飞书任务 → 关键节点自动提醒（提醒通过 `schedule-manager` 创建）

## 快速开始

### 1. 安装

重启 Ethan Agent 即可，技能会自动同步到 `~/.ethan/skills/`，`~/.ethan/work/` 目录及配置文件也会自动创建。

### 2. 配置团队

编辑 `~/.ethan/work/team.yaml`，填入你的团队信息：

```yaml
self:
  name: "张三"
  role: "前端负责人"

members:
  - name: "李四"
    role: "高级工程师"
    direction: "基础架构"
    joined: "2024-03"
    code_username: "lisi"
```

### 3. 开始使用

直接用自然语言触发：

| 场景 | 示例 |
|---|---|
| 记录事件 | 「记一下，小王今天主动排查了线上问题，10 分钟定位到根因」 |
| 文档分析 | 「帮我看看这个文档」+ 转发链接 |
| CR 统计 | 「帮我看下这周的 CR 汇总」 |
| 任务委派 | 「安排一下小王，设计插件架构，下周五前出方案」 |
| 群监控 | 「帮我监控这个群，每天扫一次」（定时任务由 schedule-manager 创建） |
| 绩效汇总 | 「帮我汇总下这季度的绩效」 |

## 数据存储

数据存储在 `~/.ethan/work/` 下：

```
~/.ethan/work/
├── team.yaml            # 团队配置（成员、项目、CR 平台映射）
├── people/              # 人员日志（唯一记录源）
│   ├── 李四.md
│   └── _self.md
├── project/             # 项目进展（链接到 people）
├── docs/                # 文档/链接收藏
├── cr-reports/          # CR 分析报告
└── reviews/             # 绩效评估草稿
```

> scene 目录隔离（work / life）、timelines.yaml、`.timeline_state.json`、exports/ 由 `schedule-manager` 负责维护，不在本技能范围内。

## 关联技能

| 技能 | 用途 |
|---|---|
| `schedule-manager` | 定时任务和时间线管理（任务提醒、周期编排、群扫描定时任务） |
| `lark-task` | 创建飞书任务和提醒 |
| `lark-im` | 发送提醒消息、扫描群消息 |
| `lark-doc` | 读取文档内容分析 |
| `lark-calendar` | 评审/对齐日历事件（可选） |
| `code-review` | CR 数据拉取与分析 |

## 设计原则

- **不做最终判定**：Agent 只提供汇总和建议，绩效打分由管理者决定
- **确认后再创建**：任务拆分、飞书任务创建，必须经用户确认
- **隐私边界**：所有数据仅存本地，不上传外部
- **通用框架**：不含任何个人/公司/团队信息，开箱即用
